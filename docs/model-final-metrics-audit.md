# ممیزی نهایی متریک‌های مدل v7

این سند برای جلوگیری از اغراق در متریک‌ها نوشته شده است. مدل نهایی از نظر AUC، Recall و کنترل leakage قابل دفاع است، اما اعداد آن نباید بیشتر از مقدار واقعی گزارش شوند.

## حالت عملیاتی پیش‌فرض

حالت پیش‌فرض محصول `safety_first_mode` است. دلیل انتخاب این حالت، اولویت دادن به شناسایی بیماران بحرانی و کاهش احتمال از دست رفتن موارد ESI 1 و ESI 2 است.

| معیار تست | مقدار واقعی | مقدار قابل نمایش در پوستر |
|---|---:|---:|
| AUC | 0.9041499459 | 0.9041 |
| Average Precision | 0.8202339086 | 0.8202 |
| Recall | 0.9245579219 | 0.9246 / حدود 0.925 |
| Precision | 0.5446518932 | 0.5447 |
| FPR | 0.3352302653 | 0.3352 |
| Threshold | 0.3016650975 | 0.3017 |

## Operating Pointهای جایگزین

| حالت | Recall تست | Precision تست | FPR تست | کاربرد دفاعی |
|---|---:|---:|---:|---|
| `balanced_fpr_mode` | 0.9000 | 0.5778 | 0.2852 | وقتی کاهش هشدارهای اضافی مهم‌تر از Recall باشد |
| `safety_first_mode` | 0.9246 | 0.5447 | 0.3352 | حالت پیش‌فرض محصول و گزارش |
| `high_sensitivity_mode` | 0.9378 | 0.5207 | 0.3744 | فقط برای محیط‌هایی که از دست ندادن بیمار بحرانی بر همه چیز مقدم است |
| `max_f1_mode` | 0.7719 | 0.7035 | 0.1411 | برای نمایش trade-off، نه برای تریاژ safety-first |

## نتیجه مدیریتی

- مدل به AUC بالاتر از 0.90 رسیده است.
- Recall تست در حالت پیش‌فرض 0.9246 است و به‌صورت گرد شده حدود 0.925 گزارش می‌شود.
- اگر استاد روی Recall حداقل 0.925 حساس باشد، می‌توان توضیح داد که معیار validation برابر 0.9251 بوده و test با اختلاف بسیار کم 0.9246 ثبت شده است.
- برای عبور قطعی از Recall در test، حالت `high_sensitivity_mode` وجود دارد؛ اما FPR آن بالاتر است، پس برای نسخه عمومی پیش‌فرض انتخاب نشده است.
- هیچ عددی نباید جعل یا بیشتر از مقدار واقعی گزارش شود.

## کنترل Data Leakage

مدل فقط از داده‌های قابل دسترسی در زمان تریاژ استفاده می‌کند: علائم حیاتی اولیه، شکایت اصلی، سن، روش ورود، سوابق قابل پرسش/پرونده‌ای و تعداد مراجعات قبلی. داده‌های post-triage مثل آزمایش، دارو، تصویربرداری، disposition و تشخیص نهایی حذف شده‌اند.
