# Stakeholder Register

## ذی‌نفعان اصلی

| ذی‌نفع | نیاز/انتظار | نوع اثر | راهبرد ارتباط |
|---|---|---|---|
| استاد درس | مشاهده مدیریت پروژه واقعی، مستندات، KPI، Agile | بسیار بالا | گزارش، پوستر، ویدئو، GitHub |
| تیم آموزشی | ارزیابی دقیق artefactها | بالا | دسترسی به board و knowledge base |
| پرستار/کادر تریاژ | ابزار سریع و قابل اعتماد | بالا در آینده | demo و بازخورد تخصصی |
| بیمار و خانواده | رسیدگی سریع‌تر و عادلانه‌تر | بالا | تاکید بر اخلاق و decision-support |
| مدیر بیمارستان | کنترل منابع و کاهش ازدحام | متوسط/بالا | KPI، FPR، roadmap |
| رئیس بیمارستان پیامبران تهران | بررسی امکان جلسه معرفی و دریافت بازخورد دامنه‌ای | بالا | ایمیل ارسال شده؛ پیگیری تا پاسخ نهایی |
| سلامت فردای تهران | بررسی محصول و امکان ارتباط برای پایلوت/بازخورد | بالا | ایمیل ارسال شده؛ پیگیری تا پاسخ نهایی |
| دانشجویان/بازدیدکنندگان پوستر | درک سریع ارزش پروژه | متوسط | پوستر بصری و demo |
| اعضای تیم | هماهنگی، یادگیری و تحویل مسئولانه | بالا | تقسیم نقش، time tracking و بازبینی Sprint |

## تحلیل Power / Interest

| گروه | قدرت | علاقه | راهبرد |
|---|---|---|---|
| استاد و تیم آموزشی | بالا | بالا | مدیریت نزدیک، مستندات کامل |
| کادر درمان | بالا | متوسط/بالا | دریافت بازخورد و اعتبارسنجی |
| مراکز درمانی مخاطب | بالا | هنوز نامشخص | ارتباط مرحله‌ای، ارائه کوتاه و پرهیز از ادعای همکاری پیش از موافقت رسمی |
| کاربران آزمایشی دانشگاهی | پایین | متوسط | demo و جمع‌آوری بازخورد |
| عموم بازدیدکنندگان | پایین | پایین/متوسط | پیام کوتاه، پوستر جذاب |

## پیام کلیدی برای هر ذی‌نفع

- استاد: پروژه فقط کدنویسی نیست؛ مدیریت، KPI، ریسک و مستندات کامل دارد.
- کادر درمان: خروجی جایگزین شما نیست، فقط هشدار کمکی و قابل توضیح است.
- مدیر بیمارستان: مدل با ورودی ناقص کار می‌کند و می‌تواند در ازدحام کمک کند.
- مراکز درمانی مخاطب: MVP آماده نمایش است؛ هدف جلسه دریافت نظر و طراحی مسیر اعتبارسنجی است، نه ادعای محصول بالینی آماده استفاده.
- بازدیدکننده پوستر: یک نمونه کاربردی AI برای نجات زمان، جان و امید.
