# برنامه مدیریت پروژه

## رویکرد

پروژه با رویکرد Agile/Scrum سبک اجرا شده است. هدف این نبود که از ابتدا محصول کامل ساخته شود؛ ابتدا مسئله و MVP تعریف شد، سپس مدل، API، UI و مستندات در sprintهای کوتاه تکمیل شدند.

## Scope

محصول فعلی: سیستم تصمیم‌یار تریاژ اورژانس شامل مدل ML، API، UI فارسی و مستندات مدیریتی.

موارد خارج از scope فعلی:

- استفاده بالینی واقعی
- اتصال به EHR واقعی
- نسخه آفلاین production
- تایید پزشکی/نهادی

## Definition of Done

یک آیتم زمانی done است که:

- خروجی قابل مشاهده یا قابل تست داشته باشد.
- در repository یا مستندات ثبت شده باشد.
- ریسک‌ها یا محدودیت‌های آن مشخص باشد.
- در صورت نیاز با سناریوی demo تست شده باشد.

## Milestoneها

| Milestone | خروجی | وضعیت |
|---|---|---|
| Project Idea | مسئله، ارزش اجتماعی و scope | انجام شده |
| ML Baseline | مدل اولیه و متریک‌ها | انجام شده |
| ML v7 | مدل safety-first، کنترل leakage، کاهش FPR و گزارش metrics | انجام شده |
| API MVP | `/health`، `/model-info` و `/predict` | انجام شده |
| Mobile UI MVP | فرم فارسی، سناریوهای demo و خروجی ریسک | انجام شده |
| Poster Package | محتوا، KPI، Burndown و ماتریس همکاری | آماده محتوا |
| Final Package | گزارش، ویدئو، board و knowledge base | آماده‌سازی شده |

## مدیریت ریسک

ریسک‌های اصلی در `docs/risk-register.md` ثبت شده‌اند. سه ریسک مهم‌تر:

- False Negative در بیمار بحرانی
- leakage یا استفاده از داده بعد از تریاژ
- عدم اعتبارسنجی با متخصص واقعی
