# برنامه مدیریت پروژه

## هدف سند

این سند روش مدیریت پروژه، نحوه برنامه ریزی، کنترل پیشرفت، ارتباطات، مدیریت ریسک و نحوه تحویل artefactهای مورد انتظار درس را مشخص می کند.

## روش انتخاب شده

روش پروژه Agile/Scrum سبک است. دلیل انتخاب این روش:

- زمان پروژه کوتاه است.
- محصول باید به سرعت به MVP برسد.
- مدل ML و نیازمندی های UI ممکن است در طول کار تغییر کنند.
- استاد روی چابکی، تحویل تدریجی و مستندسازی تاکید دارد.

این تقسیم نقش عمدا سبک و واقع بینانه است. هدف این است که همه اعضا در ارائه سهم روشن داشته باشند، اما بخش های تخصصی سنگین مانند آموزش مدل و معماری اصلی توسط عضو ارشدتر تیم مدیریت شود.

## چرخه Sprint

هر sprint یک هفته در نظر گرفته می شود.

### فعالیت های ثابت هر sprint

- Sprint planning: انتخاب user storyهای هفته
- Daily/async check: ثبت مانع ها و پیشرفت
- Sprint review: نمایش خروجی قابل مشاهده
- Sprint retrospective: ثبت درس آموخته ها
- Update poster/status: به روزرسانی وضعیت برای استاد

## Definition of Ready

یک آیتم backlog زمانی آماده ورود به sprint است که:

- هدف آن روشن باشد.
- خروجی قابل مشاهده داشته باشد.
- وابستگی اصلی آن مشخص باشد.
- معیار پذیرش داشته باشد.

## Definition of Done

یک آیتم زمانی done است که:

- کد یا مستند مربوطه تکمیل شده باشد.
- خروجی آن قابل مشاهده یا قابل بررسی باشد.
- در README یا docs در صورت نیاز منعکس شده باشد.
- ریسک یا محدودیت آن ثبت شده باشد.

## Milestoneها

| Milestone | خروجی | وضعیت |
|---|---|---|
| Project Idea | مسئله، ارزش اجتماعی، فناوری و roadmap اولیه | انجام شده / نیازمند نهایی سازی |
| ML Baseline | مدل اولیه و معیارهای ارزیابی | انجام شده |
| ML v5 | pipeline تمیز، split علمی، threshold روی validation و گزارش متریک | در حال تکمیل |
| API MVP | endpoint پیش بینی | برنامه ریزی شده |
| Mobile UI MVP | فرم ورود داده و نمایش نتیجه | برنامه ریزی شده |
| Poster Update | پوستر با roadmap، burndown و velocity | برنامه ریزی شده |
| Final Package | محصول، مستندات، AI report و lessons learned | برنامه ریزی شده |

## شاخص های کنترل پروژه

- Velocity: تعداد story pointهای تکمیل شده در هر sprint
- Burndown: کار باقی مانده نسبت به روزهای sprint
- ML Quality: AUC، Recall، Precision، FPR
- Documentation Completeness: درصد مستندات تکمیل شده
- Stakeholder Feedback: تعداد بازخوردهای دریافت شده و اعمال شده
