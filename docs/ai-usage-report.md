# گزارش استفاده از هوش مصنوعی

## اصل شفافیت

در این پروژه استفاده از ابزارهای هوش مصنوعی بخشی از فرآیند توسعه است، اما مسئولیت نهایی تحلیل، تصمیم گیری، صحت فنی و کیفیت مستندات بر عهده تیم پروژه است. خروجی های AI بدون بازبینی انسانی به عنوان نتیجه نهایی پذیرفته نمی شوند.

## اهداف استفاده از AI

- افزایش سرعت توسعه MVP
- کمک به ایده پردازی و تحلیل مسئله
- بهبود کد و مستندسازی
- کمک به طراحی معماری سیستم
- تحلیل خطاهای مدل و پیشنهاد راه حل
- آماده سازی گزارش های مدیریتی و ارائه

## ابزارهای استفاده شده

| ابزار | نوع استفاده | خروجی مورد انتظار |
|---|---|---|
| ChatGPT / Codex | تحلیل پروژه، مستندسازی، کمک به کدنویسی | مستندات، معماری، backlog، اصلاح کد |
| AI assistant قبلی | تحلیل نسخه های مدل و پیشرفت فنی | گزارش اولیه v2 تا v4 |
| کتابخانه SHAP | توضیح پذیری مدل | تصویر `shap_esi1.png` |
| مدل های ML | پیش بینی احتمال بحرانی بودن بیمار | فایل `triage_model.pkl` |

## بخش های کمک گرفته شده از AI

- تدوین مسئله و ارزش اجتماعی پروژه
- طراحی ساختار مستندات مطابق خواسته های درس
- پیشنهاد featureهای بالینی مانند `shock_index`، `pulse_pressure` و `vital_severity_score`
- تحلیل مشکلات نسخه های قبلی مدل مانند FPR بالا و استفاده نامناسب از calibration
- تنظیم رویکرد Agile و backlog
- تولید پیش نویس معماری سیستم و گزارش وضعیت

## کنترل کیفیت خروجی AI

برای جلوگیری از اتکای کورکورانه به AI، این کنترل ها در نظر گرفته شده اند:

- بررسی کد قبل از اجرا
- سنجش مدل با معیارهای کمی مانند AUC، Recall، Precision و FPR
- مقایسه نسخه های مختلف مدل
- استفاده از توضیح پذیری SHAP برای بررسی تمرکز مدل روی featureهای معنادار
- ثبت ریسک های اخلاقی و فنی
- مشخص کردن موارد TODO به جای ادعای تکمیل نشده

## ملاحظات اخلاقی

این پروژه در حوزه سلامت قرار دارد و استفاده از AI در آن باید محافظه کارانه و مسئولانه باشد:

- خروجی سامانه نباید جایگزین تصمیم متخصص شود.
- هدف مدل پشتیبانی از تصمیم است، نه تشخیص قطعی.
- خطای False Negative از نظر انسانی بسیار پرهزینه است.
- داده واقعی بیمارستانی باید با رعایت محرمانگی و حذف اطلاعات هویتی استفاده شود.
- هرگونه استفاده عملی نیازمند اعتبارسنجی بالینی و تایید نهادی است.

## محدودیت ها

- بخشی از تحلیل فنی فعلی بر اساس داده های آموزشی/ثانویه است.
- هنوز تست کاربری واقعی انجام نشده است.
- مدل باید از نظر bias نسبت به سن، جنسیت و گروه های جمعیتی بررسی شود.
- امکان hallucination در خروجی ابزارهای زبانی وجود دارد و باید با کد، داده و منبع معتبر کنترل شود.

## برنامه بهبود

- ثبت promptها و تصمیم های مهم در ادامه پروژه
- افزودن جدول کامل تعاملات AI در هر sprint
- مستندسازی خطاهای AI و اصلاحات انسانی
- اضافه کردن بخش اعتبارسنجی پزشکی/دامنه ای در صورت دسترسی به ذی نفع واقعی
