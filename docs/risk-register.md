# Risk Register

## روش ارزیابی

احتمال و اثر با مقیاس 1 تا 5 سنجیده می شوند. اولویت ریسک برابر است با:

`Risk Score = Probability x Impact`

## برنامه پاسخ اضطراری

اگر مدل به Recall هدف نرسد:

- تمرکز پروژه به MVP تصمیم یار با هشدار ریسک و توضیح پذیری تغییر می کند.
- ادعاهای فنی در ارائه محافظه کارانه می شوند.
- value پروژه روی workflow، مستندسازی، معماری و تحلیل trade-offها برجسته می شود.

اگر backend/frontend کامل نشود:

- demo با API و سناریوهای نمونه انجام می شود.
- UI به mock قابل نمایش کاهش می یابد.
- مستندات فنی و نمودارها برای دفاع از معماری آماده می شوند.

اگر بازخورد ذی نفع واقعی گرفته نشود:

- محدودیت صریحا در status report و lessons learned ثبت می شود.
- سناریوهای فرضی با احتیاط و بر اساس پروتکل عمومی تریاژ استفاده می شوند.
