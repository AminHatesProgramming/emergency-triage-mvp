# بخش آماده برای اضافه شدن به پوستر

## ابزارهای مدیریت پروژه و دانش

برای نشان دادن رویکرد Agile و مدیریت پروژه، پروژه از دو بستر جدا استفاده می‌کند:

- Work Management: GitHub Issues/Projects برای backlog، Sprintها، task owner، status و time tracking
- Knowledge Management: پوشه `docs/knowledge-base/` برای sprint notes، meeting notes، decision log، risk، AI usage و stakeholder feedback

Artifactهای قابل نمایش:

- `docs/agile-delivery-evidence.md`
- `docs/artifacts/github-issues-seed.csv`
- `docs/knowledge-base/stakeholder-feedback-log.md`

## بازخورد ذی‌نفع

در Sprint 5 فرم بازخورد داخل MVP اضافه شد. هدف تا ارائه نهایی دریافت حداقل ۵ بازخورد از دانشجوی پزشکی/پرستاری، فرد آشنا با درمانگاه یا اورژانس، دانشجوی مهندسی، کاربر عمومی و فرد آشنا با مدیریت بحران است.

چهار سؤال بازخورد:

1. آیا خروجی سیستم قابل فهم بود؟
2. آیا UI سریع و واضح بود؟
3. آیا مشخص بود سیستم جایگزین پزشک نیست؟
4. مهم‌ترین پیشنهاد بهبود چیست؟

متن کوتاه برای پوستر:

> مدیریت چابک پروژه با GitHub Issues/Projects و Knowledge Base انجام شد. هر Sprint خروجی قابل تحویل، owner، KPI و artifact مشخص دارد. در Sprint نهایی، فرم feedback داخل MVP اضافه شد تا پروژه از سطح demo فنی به سمت validation با ذی‌نفعان واقعی حرکت کند.
