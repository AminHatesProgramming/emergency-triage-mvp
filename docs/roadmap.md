# Roadmap توسعه آینده

## هدف

این roadmap برای گزارش نهایی، پوستر و ویدئو استفاده می‌شود و نشان می‌دهد پروژه بعد از MVP چه مسیر قابل دفاعی دارد.

## فاز 1: تکمیل تحویل آموزشی

بازه: تا 12 تیر

| خروجی | معیار پذیرش |
|---|---|
| پوستر نهایی A0 | شامل UI، KPI، Burndown، تیم و roadmap |
| ویدئوی 10 دقیقه‌ای | حضور هر سه عضو و Product Demo |
| گزارش نهایی | پوشش scope، risk، team، architecture، roadmap |
| board واقعی | task، assignee، time tracking |
| knowledge base | API docs، meeting notes، decision log |

## فاز 2: اعتبارسنجی دانشگاهی

بازه: تا پایان تیر

| خروجی | KPI |
|---|---|
| بازخورد کاربران آزمایشی | 35 نفر |
| بازخورد متخصص سلامت | حداقل 2 نفر |
| سناریوهای demo | 5 سناریوی تاییدشده |
| اصلاح فرم UI | حداقل 3 بهبود بر اساس بازخورد |

## فاز 3: اعتبارسنجی فنی و اخلاقی

بازه: مرداد

| خروجی | معیار |
|---|---|
| تحلیل False Negativeها | گزارش خطاهای بحرانی |
| تحلیل subgroup fairness | سن و جنسیت |
| تست robustness با ورودی ناقص | 10 سناریو |
| نسخه سبک‌تر مدل | کاهش حجم/زمان پاسخ |

## فاز 4: prototype آفلاین

بازه: شهریور

| خروجی | معیار |
|---|---|
| PWA یا mobile prototype | اجرا روی دستگاه بدون اینترنت پایدار |
| cache محلی سناریوها | demo بدون backend دائمی |
| راهنمای استفاده | مستند کوتاه برای کاربر غیر فنی |

## شرط ورود به استفاده واقعی

این پروژه تا قبل از موارد زیر نباید به عنوان ابزار عملیاتی بیمارستانی معرفی شود:

- دریافت داده واقعی بیمارستانی با مجوز
- تایید متخصص اورژانس/تریاژ
- تحلیل محرمانگی و امنیت داده
- بررسی bias و fairness
- تایید نهادی/قانونی
