## هدف مدل

مدل احتمال بحرانی بودن بیمار را پیش بینی می کند. در این پروژه، بیمار بحرانی به صورت `ESI <= 2` تعریف شده است.

مدل فقط برای پشتیبانی تصمیم طراحی شده است و نباید به عنوان جایگزین قضاوت بالینی استفاده شود.

## داده های ورودی مجاز

برای کاهش leakage، مدل v5 فقط از داده هایی استفاده می کند که در لحظه تریاژ قابل دسترسی اند:

- سن و گروه سنی
- روش ورود بیمار
- علائم حیاتی اولیه
- شکایت اصلی بیمار
- featureهای بالینی مشتق شده از علائم حیاتی

داده های زیر عمدا کنار گذاشته شده اند:

- آزمایش های بعد از تریاژ
- داروها
- imaging counts
- disposition
- diagnosisهای بعدی

## معماری مدل

مدل از ensemble چند الگوریتم tabular استفاده می کند:

- Random Forest
- Extra Trees
- XGBoost، در صورت نصب بودن وابستگی

وزن هر مدل بر اساس عملکرد validation محاسبه می شود. threshold نیز روی validation انتخاب می شود و test فقط برای گزارش نهایی نگه داشته می شود.

## متریک های اصلی v5

| معیار | دلیل اهمیت |
|---|---|
| AUC | کیفیت رتبه بندی کلی |
| Average Precision | مناسب برای داده نامتوازن |
| Recall | کاهش احتمال از دست رفتن بیمار بحرانی |
| FPR | کنترل فشار اضافی روی منابع اورژانس |
| Precision | کیفیت هشدارهای مثبت |

نتیجه تست:

| معیار | مقدار |
|---|---:|
| AUC | 0.8917 |
| Average Precision | 0.7981 |
| Recall | 0.9194 |
| Precision | 0.5275 |
| F1 | 0.6704 |
| FPR | 0.3572 |
| Specificity | 0.6428 |
| Threshold | 0.3061 |

هدف AUC با حاشیه مناسب محقق شده است. Recall با مقدار 0.9194 بسیار نزدیک به هدف 0.92 است و FPR نسبت به نسخه قبلی گزارش شده، یعنی 0.487، کاهش قابل توجهی داشته است.

## تصمیم های قابل دفاع برای ارائه

- threshold روی validation انتخاب می شود تا test آلوده نشود.
- تمرکز روی Recall به دلیل هزینه انسانی False Negative است.
- featureهای بعد از تریاژ حذف شده اند تا مدل در سناریوی واقعی قابل دفاع باشد.
- خروجی مدل همراه با توضیح و هشدار اخلاقی ارائه می شود.
