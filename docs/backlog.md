# Product Backlog و Sprint Plan

## هدف محصول

به عنوان پرسنل اورژانس، کاربر باید بتواند اطلاعات اولیه بیمار را سریع وارد کند و یک پیشنهاد تریاژ قابل توضیح دریافت کند تا احتمال از دست رفتن بیمار بحرانی کاهش یابد.

## مقیاس اولویت

- P0: ضروری برای MVP
- P1: مهم برای کیفیت ارائه
- P2: قابل انجام در صورت زمان اضافه

## Sprint 0: آماده سازی و جهت دهی

هدف: تثبیت موضوع، خواندن الزامات استاد، ساخت اسکلت repository و مستندات پایه.

خروجی ها:

- README پروژه
- معماری اولیه
- گزارش استفاده از AI
- risk register
- backlog
- status report اولیه

## Sprint 1: MVP مدل و API

هدف: تبدیل مدل فعلی به سرویس قابل فراخوانی.

آیتم های پیشنهادی:

- US-03
- US-05
- US-06
- US-12

خروجی قابل نمایش:

- endpoint پیش بینی
- پاسخ JSON برای بیمار نمونه
- نمایش معیارهای مدل

## Sprint 2: رابط کاربری موبایل

هدف: ساخت UI ساده برای ورود اطلاعات و نمایش تصمیم.

آیتم های پیشنهادی:

- US-01
- US-02
- US-04
- US-09

خروجی قابل نمایش:

- فرم mobile-first
- نمایش probability، label و explanation

## Sprint 3: کیفیت، پوستر و ارائه

هدف: آماده سازی خروجی قابل دفاع برای استاد.

آیتم های پیشنهادی:

- US-07
- US-08
- US-11
- US-13

خروجی قابل نمایش:

- burndown و velocity
- lessons learned
