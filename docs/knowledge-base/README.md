# Emergency Triage Knowledge Base

این پوشه نقش Knowledge Management System پروژه را دارد. هدف آن این است که استاد فقط چند فایل گزارش نبیند، بلکه مسیر تصمیم‌گیری، یادگیری، جلسات، بازخوردها و تغییرات پروژه قابل ردگیری باشد.

## صفحات اصلی

| صفحه | مالک | هدف |
|---|---|---|
| `sprint-notes.md` | محمدامین | ثبت هدف و خروجی هر Sprint |
| `meeting-notes.md` | محدثه | ثبت صورتجلسات کوتاه و تصمیم‌ها |
| `technical-decisions.md` | محمدامین | ثبت تصمیم‌های معماری، مدل و API |
| `stakeholder-feedback-log.md` | محمدرضا | ثبت بازخورد کاربران و ذی‌نفعان |
| `team-playbook.md` | همه | توضیح نقش‌ها و متن آماده برای ارائه اعضا |
| `safety-layer-v1-fa.md` | محمدامین | ثبت منطق لایه ایمنی سن‌محور و کنترل داده‌های پرت |
| `../model-release-scenario-audit-fa.md` | محمدامین | ممیزی held-out، ورودی ناقص، سناریوهای مرزی و محدودیت اعتبارسنجی |

## قاعده نگهداری

- هر تغییر مهم باید یا در `technical-decisions.md` یا در `decision-log.md` ثبت شود.
- هر Sprint باید یک خروجی قابل مشاهده داشته باشد.
- هر بازخورد کاربر باید با تاریخ، نوع ذی‌نفع، نکته مثبت، نگرانی و اقدام بعدی ثبت شود.
- اگر ابزار بیرونی مثل Notion یا Google Docs ساخته شد، لینک آن باید در همین README اضافه شود.
- شواهد مدل باید بین «ارزیابی داخلی»، «بازخورد کاربردپذیری» و «اعتبارسنجی بالینی» تفکیک صریح داشته باشند.
