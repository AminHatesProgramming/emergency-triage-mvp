# Team Playbook

این سند کمک می‌کند نقش اعضای تیم در ارائه واقعی و قابل دفاع دیده شود، بدون اینکه هم‌تیمی‌ها وارد توضیحات خیلی تخصصی شوند.

## محمدامین پورمند

Role: Project Lead / ML & System Architect

چیزهایی که توضیح می‌دهد:
- چرا داده‌های post-triage حذف شدند.
- چرا Recall نسبت به Precision اولویت دارد.
- معماری API-first و mobile-first چیست.
- مدل چرا decision-support است نه جایگزین متخصص.

## محمدرضا آرمان پور

Role: Project Control & Stakeholder/KPI Coordinator

چیزهایی که توضیح می‌دهد:
- KPIهای پروژه: AUC، Recall، FPR، data completeness، feedback count.
- چرا trade-off بین Recall و FPR در اورژانس قابل دفاع است.
- risk register و mitigationهای اصلی.
- برنامه جمع‌آوری بازخورد از 5 تا 10 نفر.

متن آماده:

> نقش من این بود که پروژه فقط فنی دیده نشود. KPIها، ریسک‌ها، burndown و برنامه feedback را دنبال کردم تا نشان دهیم محصول به سمت استفاده واقعی حرکت کرده است.

## محدثه حاتمی کیا

Role: UI/Documentation & QA Coordinator

چیزهایی که توضیح می‌دهد:
- UI موبایل چطور برای تریاژ سریع طراحی شده است.
- سناریوهای تست بحرانی، متوسط، کم‌داده و قلبی.
- چرا disclaimer و توضیح خروجی مهم است.
- مستندات و Knowledge Base چطور مرتب شده‌اند.

متن آماده:

> نقش من بررسی قابل فهم بودن UI، تست سناریوهای نمونه و مرتب‌سازی مستندات بود. تمرکز داشتم خروجی برای کاربر غیرمتخصص هم سریع و واضح باشد.
