# طرح ارائه پروژه

## اسلاید 1: عنوان

سیستم هوشمند پشتیبان تصمیم‌گیری تریاژ اورژانس

اعضا: محمدامین پورمند، محمدرضا آرمان پور، محدثه حاتمی کیا

## اسلاید 2: مسئله

- ازدحام اورژانس و فشار تصمیم‌گیری
- وابستگی تریاژ به تجربه فردی و کیفیت داده لحظه‌ای
- هزینه انسانی False Negative

## اسلاید 3: راه‌حل پیشنهادی

- مدل ML برای تخمین احتمال بحرانی بودن بیمار
- API و UI قابل demo
- decision-support، نه جایگزین متخصص

## اسلاید 4: رویکرد مدیریت پروژه

- Agile/Scrum سبک
- backlog، sprint، risk register و status report
- مستندسازی AI usage و تصمیم‌های فنی

## اسلاید 5: داده و کنترل leakage

- استفاده از اطلاعات triage-time
- حذف labs، meds، imaging، disposition و تشخیص‌های بعدی
- حذف متغیرهای حساس تا قبل از fairness review

## اسلاید 6: مدل v7

- XGBoost انتخاب‌شده بر اساس validation
- اصلاح واحد دما
- سابقه‌های بالینی EHR-safe
- missingness features برای ورودی ناقص

## اسلاید 7: نتایج

| معیار | مقدار |
|---|---:|
| AUC | 0.9041 |
| Recall | 0.9246 |
| Precision | 0.5447 |

نکته ارائه: safety-first انتخاب شده چون در تریاژ، از دست دادن بیمار بحرانی خطرناک‌تر از بررسی اضافه است.

## اسلاید 8: Demo

- سناریوی بحرانی
- سناریوی متوسط
- ورودی sparse با ۴ آیتم
- نمایش data completeness و missing fields

## اسلاید 9: اخلاق و مسئولیت اجتماعی

- عدم جایگزینی متخصص
- توجه به تاب‌آوری درمانی کشور
- شفافیت در محدودیت‌ها
- نیاز به اعتبارسنجی بالینی

## اسلاید 10: نقش اعضا و جمع‌بندی

- محمدامین: معماری، مدل و API
- محمدرضا: شاخص‌ها، ریسک‌ها و ارزش اجتماعی
- محدثه: UI، مستندات و QA

جمع‌بندی: پروژه یک MVP قابل اجرا، مستند و قابل دفاع برای درس مدیریت پروژه فناوری اطلاعات است.
