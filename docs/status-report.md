# گزارش وضعیت پروژه

تاریخ آخرین به روزرسانی: 2026-06-01

## خلاصه اجرایی

پروژه «سیستم هوشمند پشتیبان تصمیم گیری تریاژ اورژانس» در مرحله آماده سازی MVP و مستندسازی مدیریتی قرار دارد. بخش ML از نسخه آزمایشی v4 به pipeline حرفه ای v5 ارتقا پیدا کرده است؛ در v5 انتخاب threshold از test جدا شده، featureهای بعد از تریاژ حذف شده اند و گزارش متریک قابل استناد تولید می شود. backend اولیه نیز با FastAPI آماده شده است. تمرکز sprint بعدی ساخت رابط mobile-first برای demo است.

- تولید نمودارهای تحلیل مدل و گزارش JSON متریک ها

- برجسته کردن مسئولیت اجتماعی، اخلاق AI و خدمت به تاب آوری درمانی کشور

## پیشرفت فنی

نسخه های مدل بر اساس گزارش فعلی:

| نسخه | AUC | نکته اصلی |
|---|---:|---|
| v2 | 0.8467 | استفاده نامناسب از calibration و FPR بالا |
| v3 | 0.8467 | افت عملکرد به دلیل تنظیم نامناسب XGBoost |
| v4 | حدود 0.87+ | حذف calibration و بهبود ensemble |
| v5 | 0.8917 | split جداگانه train/validation/test، کنترل leakage، انتخاب XGBoost |

متریک های تست v5:

| معیار | مقدار |
|---|---:|
| AUC | 0.8917 |
| Average Precision | 0.7981 |
| Recall بیماران بحرانی | 0.9194 |
| Precision | 0.5275 |
| F1 | 0.6704 |
| FPR | 0.3572 |
| Specificity | 0.6428 |
| Threshold | 0.3061 |

نکته قابل ارائه: Recall کمی پایین تر از هدف قراردادی 0.92 و در عمل بسیار نزدیک به آن است؛ در مقابل AUC و FPR به شکل معناداری بهتر شده اند. در صورت اصرار روی Recall حداقل 0.92، threshold می تواند اندکی محافظه کارانه تر شود.
