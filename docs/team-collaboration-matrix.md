# ماتریس همکاری تیم و ثبت زمان

این جدول جمع‌بندی برآورد تیم از زمان صرف‌شده برای MVP است. اعداد با Time Tracking و خروجی‌های ثبت‌شده تطبیق داده شده‌اند، اما تایم‌شیت حقوقی محسوب نمی‌شوند.

## نقش‌ها

| عضو | نقش | مسئولیت‌های قابل ارائه |
|---|---|---|
| محمدامین پورمند | Project Lead / ML & System Architect | مدل، backend، معماری، GitHub و یکپارچه‌سازی |
| محمدرضا آرمان پور | Project Control & Metrics Coordinator | KPIها، ریسک‌ها، ارزش اجتماعی، trade-offهای مدیریتی |
| محدثه حاتمی کیا | UI/Documentation & QA Coordinator | user flow، چک‌لیست QA، مستندات، سناریوهای demo |

## Time Tracking پیشنهادی

| عضو | تحلیل/برنامه‌ریزی | پیاده‌سازی | مستندات | تست/ارائه | جمع |
|---|---:|---:|---:|---:|---:|
| محمدامین پورمند | 7h | 26h | 8h | 6h | 47h |
| محمدرضا آرمان پور | 4h | 2h | 8h | 4h | 18h |
| محدثه حاتمی کیا | 4h | 5h | 9h | 5h | 23h |
| جمع کل | 15h | 33h | 25h | 15h | 88h |

## توزیع وظایف

| deliverable | محمدامین | محمدرضا | محدثه |
|---|---:|---:|---:|
| انتخاب مسئله و scope | 50% | 25% | 25% |
| مدل ML و metrics | 85% | 10% | 5% |
| API و integration | 90% | 5% | 5% |
| UI و سناریوهای demo | 55% | 10% | 35% |
| مستندات مدیریتی | 45% | 30% | 25% |
| پوستر و ارائه | 35% | 30% | 35% |

## نکته ارائه

محمدرضا می‌تواند KPIها، ریسک‌ها و trade-off بین Recall/FPR را توضیح دهد. محدثه می‌تواند flow کاربر، سناریوهای تست و بخش اخلاق AI را ارائه کند. این تقسیم باعث می‌شود مشارکت تیمی واضح باشد بدون اینکه اعضا مجبور شوند وارد جزئیات پیچیده ML شوند.
