# راهنمای تصویربرداری نهایی Jira و Notion

این راهنما تصاویر لازم برای اثبات استفاده واقعی از ابزارهای مدیریت پروژه و مدیریت دانش را مشخص می‌کند. تصاویر باید خوانا، بدون اطلاعات شخصی حساس و مطابق وضعیت واقعی پروژه باشند.

تصاویر نهایی در مسیر <span dir="ltr">docs/artifacts/project-tool-screenshots/</span> ذخیره می‌شوند. چون Jira و Notion فقط در نشست احراز هویت‌شده نمایش کامل دارند، ثبت تصویر باید در همان نشست کاربری انجام شود؛ داده‌های تصویر از روی وضعیت واقعی ابزارها برداشت می‌شوند و تصویر شبیه‌سازی‌شده جایگزین مدرک رابط نیست.

## تنظیمات عمومی

- Zoom مرورگر حدود ۸۰ تا ۹۰ درصد باشد تا عنوان پروژه و محتوای اصلی همزمان دیده شوند.
- اعلان‌های موقت، Quickstart و پنجره‌های اضافه بسته شوند.
- عنوان پروژه یا دانشنامه در هر تصویر قابل مشاهده باشد.
- ایمیل، شماره تماس، هویت ذی‌نفعان و اطلاعات خصوصی از کادر خارج یا پوشانده شوند.
- کارهای باز <span dir="ltr">EMD-60</span> و <span dir="ltr">EMD-61</span> به‌اشتباه Done نمایش داده نشوند.

## کنترل نهایی

- هر تصویر فقط یک ادعای مشخص را ثابت کند.
- تصاویر تزئینی، تکراری یا دارای اطلاعات حساس حذف شوند.
- وضعیت‌ها با Jira، Notion و GitHub هم‌خوان باشند.
- پس از ثبت، تصاویر در پوشه Evidence مخزن و صفحه Evidence Index لینک شوند.
- نام و تعداد فایل‌ها با فهرست این راهنما تطبیق داده شود؛ خروجی نهایی شامل پنج تصویر Jira و شش تصویر Notion است.
